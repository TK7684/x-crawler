- @mattpocockuk: By far the strangest TypeScript trick I've ever come across is the Loose Autocomplete trick.

No idea who discovered this, but it's become widely used by OSS wizards.

It's now even used by React's types to type attributes like aria-role.
- @mattpocockuk: I don't _really_ know why this works, but here's my best guess:

Without the & {}, TypeScript collapses 'small' | 'large' | string into just 'string'. It does this IMMEDIATELY. If you hover over the first example of Size, you'll see that it's just 'string'.
- @mattpocockuk: But WITH the & {}, it doesn't collapse the union immediately. It does later (when it's used).

If you hover the second Size type, you'll see it's still 'small' | 'large' | (string & {}), not 'string'.
- @mattpocockuk: So that means there's a window that the union hasn't been collapsed where autocomplete can be provided.
- @mattpocockuk: The earliest sighting I've seen is a suggestion in a thread by 
@SeaRyanC
- @cefzanella: What's your opinion on this kind of thing?

I'm very against this type of trick, because you're making your code confusing and not adding much value out of it.

Id I see "'something' | string" during a code review, my first instinct is to reject it because it surely looks wrong.
- @mattpocockuk: I'd advise capturing it in a type helper:

// Lets you pass string while also giving autocomplete
// for T
type LooseAutocomplete<T> = T | (string & {});
